Finance Dashboard System Documentation
========================================

This project is split into two main parts:
- `backend` for API logic, authentication, and database operations.
- `frontend` for the dashboard user interface and protected pages.

Backend Setup
-------------
1. Create the Laravel project in the `backend` folder:
   - `composer create-project laravel/laravel backend`
2. Configure the database in `.env`.
3. Create models and migrations as needed.
   - Example: `php artisan make:model FinancialRecord -m`
4. Run migrations:
   - `php artisan migrate`

Database Design
---------------
- Use foreign key constraints with `nullOnDelete` for category relationships to avoid accidental data loss.
- Use soft deletes on financial records to allow recovery and avoid permanent deletion.
- Separate filtering/search logic into a dedicated search class or service for cleaner controllers and better scalability.
- Use seeders to insert default data for categories and admin users.

API and Authentication
----------------------
- Use Laravel Sanctum for API authentication.
- Use a single authentication guard with role-based middleware.
- Protect API routes based on user roles:
  - Authenticated users can access dashboard overview.
  - `analyst` and `admin` can access financial records and category listing.
  - `admin` can create, update, delete records and manage users.

Example route structure:

    Route::middleware(['auth:sanctum'])->group(function () {
        Route::get('/dashboard', ...);
    });

    Route::middleware(['auth:sanctum', 'role:analyst,admin'])->group(function () {
        Route::get('/financial-records', ...);
        Route::get('/financial-records/{id}', ...);
        Route::get('/categories', ...);
    });

    Route::middleware(['auth:sanctum', 'role:admin'])->group(function () {
        Route::post('/financial-records', ...);
        Route::put('/financial-records/{id}', ...);
        Route::delete('/financial-records/{id}', ...);
        Route::get('/user-management', ...);
        Route::put('/user-management/{id}', ...);
        Route::delete('/user-management/{id}', ...);
    });

Authentication API
------------------
- `POST /api/register` to create a new user.
- `POST /api/login` to log in.

Example registration body:

    {
      "name": "abhinav",
      "email": "abhinav@gmail.com",
      "password": "1234567890",
      "password_confirmation": "1234567890"
    }

- Validate input properly and use appropriate HTTP status codes.

Frontend Setup
--------------
1. Create the React project using Vite in the `frontend` folder:
   - `npm create vite@latest`
2. Install dependencies:
   - `npm install`
3. Use `AuthContext` to protect pages that require login.
4. Protect routes based on user role and authentication status.

Frontend Notes
--------------
- Pages like `/dashboard` should be protected with authentication.
- Role-based access should be enforced in the frontend for admin, analyst, and viewer roles.

This documentation is intended to help understand the project structure, setup, and main implementation details.
